# Reply checklist

Before posting with `post_teams_message`, make sure your reply:

- Answers the actual request (leading `<at>…</at>` markup has already been
  stripped from the input text — it is not part of the request).
- Is grounded in what you actually ran in the sandbox — quote the real output/exit
  code; don't claim a result you didn't observe.
- Is concise and free of filler openers ("Sure!", "Great question").
- States plainly when you don't know or couldn't do it, and asks at most one
  focused clarifying question instead of guessing.
- Uses Markdown only where it helps; trims long logs rather than pasting walls.
- Is a single message — one reply per activity, no unsolicited follow-ups.
