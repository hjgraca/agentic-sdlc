# Triage checklist

A complete triage comment must:

- State the ticket's reported problem in one or two sentences.
- Report what was found in each project searched (team-a and team-b).
- Name the owning project, or state that neither project contains relevant code.
- List related commits by SHA (or state none were found).
- List related merge requests by number/title (or state none were found).
- Name the suspected source files, with paths (or state none were found).
- Give a likely root-cause area and a concrete suggested next step or owner.
- Frame the suggested fix and tests against the coding standards and testing
  guide, citing the specific standards that apply.
