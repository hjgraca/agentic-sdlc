# Reply checklist (interview)

Before posting, make sure your message:

- Asks exactly ONE question (during the interview), or publishes the spec — not
  both at once.
- Doesn't re-ask something already answered earlier in this conversation.
- Is concise and reads like a teammate — no filler ("Great question!").
- Reflects ALL participants when several answered; names disagreements instead
  of silently choosing.
- Uses Slack mrkdwn lightly (`*bold*`, `` `code` ``, bullets); for the spec, a
  fenced Markdown block is fine.
- Ends the turn after posting — you'll be woken by the next reply. Never assume
  you can ask several questions in one run.
