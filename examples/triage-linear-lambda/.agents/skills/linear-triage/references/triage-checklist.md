# Triage checklist

A complete triage comment must:

- State the reported issue's title and the comment that triggered the triage.
- Identify the current state, priority level, and existing labels (or note that
  there are none).
- Report which label(s) were applied (or explain why no label was applied).
- Report the new assignee (or explain why the issue was left unassigned).
- Give a likely root-cause hypothesis based on the issue's title and description.
- Provide a concrete suggested next step (with an owner if one was identified).
- Be written in a tone appropriate for a technical team: direct, specific, and
  actionable — no filler phrases.
