# Triage checklist

A complete triage comment must:

- State the issue's reported problem in one or two sentences.
- Report what was searched and found in the repo (and any extra repos listed).
- Name the likely root-cause area, or state that relevant code could not be
  located.
- Name the suspected source files, with paths (or state none were found).
- List related pull requests by number/title (or state none were found).
- Give a concrete suggested next step or owner.
- Frame the suggested fix and tests against the repo's conventions
  (CONTRIBUTING.md, AGENTS.md/CLAUDE.md), citing the specific conventions that
  apply.
